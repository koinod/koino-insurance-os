## 2026-05-04

- **FLOOR-AUDIT** — Audit of the rep work surface + fix wave for everything broken or stub.
  - **Critical: dial-loop infinite recursion fixed.** `repflowCall` (page-platform.jsx) used to delegate to `repflowDial` (page-tenant.jsx), which fell back to `repflowCall` when Twilio token failed → call-stack overflow on every dial when Twilio creds aren't set. Restructured so `repflowCall` is the public cascade entry (Twilio softphone → desktop helper → tel:) and `repflowDialTwilio` is a Twilio-only helper returning a boolean. `repflowDial` kept as a backwards-compat alias. Files: `page-platform.jsx`, `page-tenant.jsx`.
  - **Floor AutodialerPill is now functional.** Previously the ON/PAUSE/STOP buttons only flipped localStorage — no leads were dialed. Now Start builds the rep's queue (their assigned + unassigned rows, ranked by SLA + score) and dispatches `autodial:start` to AutoDialBar; Pause / Resume / Stop dispatch the matching events. AutoDialBar grew listeners for `autodial:pause` / `autodial:resume` / `autodial:stop`. AutoDialBar now uses the lead's real `phone` field when present, only falling back to the synthetic test number for demo seeds. Files: `page-floor.jsx`, `page-autodialer.jsx`.
  - **InCall stub buttons wired.** Mute toggles state + calls `connection.mute()` on the Twilio Voice SDK active connection. Hold pauses the call timer and toggles state. SMS opens the new compose modal. Schedule SOA opens the new appointment modal pre-set to 15-min SOA mode. Send app link copies a placeholder e-app URL to clipboard and logs a follow-up task. Files: `page-queue.jsx`.
  - **NEW: SMS automation.** `/api/twilio-sms.js` posts to Twilio Programmable Messaging via REST; graceful 503 with `missing: [TWILIO_*]` when env unset. New `<SmsComposeModal/>` with 4 templates (follow-up nudge, SOA reminder, missed-you, thanks-next), 1600-char counter, recents log to `AppData.SMS_LOG`. Trigger from queue rows (new SMS button), in-call panel, and any future site via `window.smsCompose(lead, phone)`. Files: `api/twilio-sms.js` (new), `page-floor-actions.jsx` (new).

Triggers: Schedule SOA button on queue rows, in-call panel; `window.scheduleSOA(lead)` / `window.bookAppointment(lead)` for any other surface. Files: `page-floor-actions.jsx`.
  - **NEW: Live + post-call transcription via Whisper.** `/api/transcribe.js` accepts both `{ audio_url }` (recording webhooks) and multipart file uploads (live mic chunks); calls OpenAI Whisper (`whisper-1`, `gpt-4o-transcribe` if `OPENAI_TRANSCRIBE_MODEL` set); returns verbose JSON with segments. `<LiveTranscriber/>` captures the rep's mic + the Twilio Voice SDK's remote audio track via Web Audio API, mixes with `MediaStreamAudioDestinationNode`, slices into 5-second `MediaRecorder` chunks, posts each to `/api/transcribe`, and appends labeled segments. Falls back to mic-only when no Twilio call is active. The fake "Live transcript" panel in InCall replaced with the real component. `<PostCallTranscript/>` polls vault_artifacts for finished recording transcripts. The Twilio recording webhook (`api/twilio-recording.js`) now fires the transcription job after persisting the artifact (uses `req.waitUntil` when available; basic-auth-aware fetch for Twilio recording URLs). Graceful 503 if `OPENAI_API_KEY` not set — UI shows the env-var setup hint. Files: `api/transcribe.js` (new), `api/twilio-recording.js`, `page-transcriber.jsx` (new).
