- **LOCAL-AUTODIAL-CADENCE** — What the best autodialers do that Phone Link / Continuity didn't: the cadence engine. Web app can't programmatically press "Call" inside Phone Link (Microsoft doesn't expose an API), but most of the autodialer's value is in *what happens between dials*, not the dial trigger itself.
  - **New `lib/dial-rules.js`** — single source of truth for dial gates + cadence. Loaded after `lib/me.js` so all surfaces share the same checks.
  - **TCPA dial-window enforcement.** State → IANA timezone map (50 + DC), federal 8am-9pm window enforced at the lead's local time. State overrides for FL (Sunday 9am), MS/AR/LA/OK (no-Sunday). `window.canDialNow(lead)` returns `{ ok, reason, untilLocalHour }`. Autodialer skips out-of-window leads, logs `out_of_window` outcome, advances after 600ms.
  - **Cooldown between dials.** 5-minute floor between attempts on the same lead — prevents accidental redial spam if a rep hammers the dial button. Stored in sessionStorage. `window.dialCooldown(leadId)` returns ms remaining; `window.markDialAttempt(leadId)` is fired by the autodialer before each dial.
  - **Disposition-driven retry queue.** Autodialer outcomes now schedule auto-redials per cadence: `no_answer` → 2h, `voicemail` → 24h, `callback` → manual, `appointment` / `not_interested` / `no_contact_info` → terminal. `window.scheduleRedial(lead, outcome)` writes to localStorage; `window.dueRedials()` returns leads ripe to re-enter the queue. Background interval fires `redial:due` events every 60s for the autodialer to pull in.
