### 🔒 Security finding — `/api/agents/issue-token`
Anyone in the world can `POST /api/agents/issue-token` with empty body and receive a valid `rfk_*` enrollment token + the install command. Source confirms it is unauth by design (`api/agents/issue-token.js`).

Risk:
- DoS-fill `agent_install_tokens` table
- Mint tokens an attacker can use with the install script to enroll a phantom host the operator never asked for
- Returns the (already-public) anon key — low marginal risk

**Recommend**: gate behind owner JWT (same pattern as `/api/invites/create`), or rate-limit + tie to a session cookie. At minimum a `Origin` check.

### RLS verification
Direct PostgREST hit with anon key on `/rest/v1/reps?limit=3` → `[]`. RLS is enforcing, not leaking unauth data. ✓
